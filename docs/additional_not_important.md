# Умная сортировка автокомплита (идея, не в работе)

## Цель

Сделать меню автокомплита (Ctrl+Space) в Theia IDE контекстно-умным: поднимать наверх те кандидаты, которые наиболее релевантны тому, что пишет разработчик прямо сейчас, а не просто в алфавитном порядке.

## Концепция

LSP уже формирует валидный список кандидатов. Задача — не добавлять новые элементы, а **переставить порядок** уже существующих через reranker.

```
LSP → список валидных кандидатов
  → Qwen3-Reranker-0.6B (/v1/rerank, порт 8030)
       query     = контекст у курсора (текущая строка, enclosing-функция, languageId)
       documents = label + detail каждого кандидата
       scores    → новый sortText + ★ у топ-3
  → меню с умным порядком
```

## Ключевые решения

- **Модель** — Qwen3-Reranker-0.6B, уже поднята в NES-стеке (порт 8030). Новой инфраструктуры не нужно.
- **Валидность** от LSP, **порядок** от reranker. Мусор внести нельзя.
- **Вызов по требованию** (Ctrl+Space), не на каждый keystroke.
- **Fail-open** — reranker недоступен → список LSP как есть, меню не блокируется.
- **Только код** — проза не трогается.
- **Кэш** по (тип получателя + префикс) — пока дотипываешь тот же префикс, переиспользуем ранжирование.

## Точка интеграции

Обёртка `monaco.languages.registerCompletionItemProvider` как `FrontendApplicationContribution` — перехватывает результат провайдеров и прогоняет через reranker перед возвратом в меню.
